# Tcom Interaction Design Specifications (v2.0)

本规范定义了 Tcom 应用程序中四种核心交互组件的视觉样式、交互行为与技术实现标准。

---

## 1. Toast (消息通知)
用于提供轻量级的、不干扰操作的反馈信息。

### 视觉特性
- **位置**: 页面顶部居中，距离顶部 26px。
- **背景**: `bg-[#252526]` (VSCode Sidebar 背景色)。
- **边框**: 2px 实线，颜色根据类型区分：
  - `success`: `#22c55e` (Green-500)
  - `error`: `#ef4444` (Red-500)
  - `info`: `#3b82f6` (Blue-500)
  - `warning`: `#eab308` (Yellow-500)
- **阴影**: 对应颜色的扩散投影，如 `0 0 15px rgba(34, 197, 94, 0.2)`。
- **字体**: `Fira Sans` (或其他 Sans 字体)，14px，Medium。

### 交互行为
- **自动消失**: 默认 1-3 秒后向上滑动消失。
- **堆叠逻辑**: 最多显示 3 条，超出的旧消息将被强制移出。
- **动画**: `translateY` 向上偏移并配合 `opacity` 与 `scale` 的平滑过渡。

---

## 2. Confirm (确认框)
用于高风险或关键操作的二次确认，基于 `UpdateDialog` 视觉进化。

### 视觉特性
- **背景**: `bg-[#252526]`，带 `backdrop-blur-sm` 的 `black/60` 遮罩。
- **头部 (Header)**: `bg-[#2d2d2d]`，标题为 `uppercase tracking-wider` 样式的 12px 粗体文字。
- **内容区**:
  - **标题 (Title)**: 14px 粗体，位于图标右侧。
  - **描述 (Description)**: 12px 浅灰色 (`#969696`)。
- **图标 (Icon)**:
  - `Warning/Danger`: `AlertCircle` (Red-500)
  - `Question`: `Info` (Blue-500)
- **按钮 (Buttons)**:
  - **确定**: `bg-[#0e639c]` 或 `bg-[#a1260d]` (危险操作)，带 200ms 的悬停颜色渐变。
  - **取消**: `bg-[#3c3c3c]` 透明背景，边框为虚线或无边框。

### 行为逻辑
- **Promise 驱动**: 使用 `await confirm({ ... })` 异步调用。
- **焦点控制**: 进入时默认聚焦在“取消”按钮上，防止回车误触危险操作。

---

## 3. Dialog (业务对话框)
承载复杂输入逻辑的功能层。

### 视觉特性
- **宽度**: 常用规格为 450px 或 600px。
- **布局**: 标准三段式结构（Header, Content, Footer）。
- **表单控件**: 统一使用 `bg-[#3c3c3c]` 输入框，`focus:border-[var(--vscode-selection)]`。
- **排版**: 标签使用 11px `uppercase`，输入框文字 13px。
- **投影**: `shadow-2xl`。

### 交互准则
- **点击遮罩**: 默认不关闭，必须点击“取消”或“关闭”图标以防丢失未保存的表单数据。
- **快捷键**: `Esc` 关闭，`Enter` 提交（多行文本框除外）。

---

## 4. Context Menu (右键菜单)
提供对象的快速操作选项。

### 视觉特性
- **背景**: `bg-[#252526]`，带微量边框 `#454545`。
- **列表项 (Item)**: 
  - **默认状态**: 12px 文字，带图标。
  - **悬停状态**: `bg-[#094771]` (VSCode 选择色)，文字保持可见。
- **分割线**: 1px 实线 `#3c3c3c`。
- **投影**: `shadow-lg`。

### 交互准则
- **智能定位**: 自动检测页面边缘，防止菜单内容超出屏幕范围。
- **点击穿透**: 点击菜单外任意区域立即关闭。
